    Visit Date
      • Denominator is achieved when a Progress Note is signed.
      • Numerator is achieved when a Summary of Care file is sent successfully via Direct Message and its status is received as either Delivered / Not Delivered.
      • Actions required to achieve this measure are as follows:
        • Navigate to Clinical Summary and export Continuity of Care Document from “Export C-CDA” button.
        • Select the Clinical Visit for which Referral Note or Continuity of Care Document is required to sent.
        • Select Data Elements to be made part of the file and click on Transmit Referral Note / CCD.
        • Enter a valid Direct address, add message content and click on Transmit button. The Referral Note / CCD will be transmitted.

              Note: Multiple denominators and numerators can be achieved for a patient in a reporting period.
      • Denominator is achieved when a new, referred and transitioned patient.
      • Numerator is achieved when a Summary of Care/CCDA is imported in the system.
      • Summary of care refers to Referral Note and Continuity of Care Document.
      • If a summary of care is received and not incorporated in the system, numerator will not increment.
      • Actions required to achieve this measure are as follows:
        • Navigate to Batch Module, choose & upload New Patient’s Summary of Care/CCDA, select a provider and click Import button to add new patient in application.
        • This would achieve both Denominator & Numerator at the same time for this measure.
        • The other work flow is while adding new patient through Add Patient or Quick Add Patient screen where there is a mandatory drop-down field named CCDA available.
        • If user select No under this drop-down, this measure will not be applicable for this patient.
        • If user selects Yes and adds the patient after selecting Yes, then the denominator will be achieved but user would be unable to achieve numerator.
        • The processes of passing this measure for that patient would be to Delete that patient and add it again by importing it through Batch module.
          OR
        • The user can export patient’s CCDA, mark patient Inactive & upload that CCDA as a new patient.

              Note: The numerator will be counted once in a reporting period for a patient.
